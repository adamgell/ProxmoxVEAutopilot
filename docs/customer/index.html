Initial Setup Deployments Readiness Docs

Initial Setup

The Proxmox console installer performs most of the stack creation. It runs the environment through Foundation, Bootstrap, and Operational phases.

Main customer takeaway: the user mostly runs one installer on the Proxmox VE host. The script prepares Proxmox, creates the Ubuntu controller, starts the application stack, prepares media, coordinates the Windows build host, and publishes readiness into the web UI.

Foundation

Prepares Proxmox and creates the controller.

  • API token, role, and ACLs
  • Storage, bridge, node, and IP detection
  • Ubuntu controller VM
  • Controller runtime health

Bootstrap

Prepares deployment inputs.

  • Windows installation media
  • VirtIO driver media
  • Setup state publishing
  • Blank deployment templates

Operational

Makes the stack ready for users.

  • Controller health verified
  • Build artifacts promoted
  • ISO storage updated
  • /setup reports ready
1. Run installerinstall-proxmox-ve.sh
2. FoundationProxmox access and controller VM
3. RuntimeWeb UI, API, MCP, Postgres, monitor
4. BootstrapWindows and VirtIO media
5. Build hostSource-built MSI, WinPE, CloudOSD
6. Ready/setup operational

Windows Deployment Workflows

After setup is ready, users choose a deployment path based on whether they are deploying workstations or servers.

Workstations

Client Windows deployments use OSDCloud.

  • Client Windows image deployment
  • AutopilotAgent first-boot setup
  • Hardware hash capture
  • Optional Intune/Autopilot upload
  • Optional domain verification

Servers

Windows Server deployments use OSDeploy.

  • Server image deployment
  • Base readiness verification
  • Optional domain join
  • File server role
  • Isolated domain controller, MECM prereq, or Lab in a Box
1. Choose pathWorkstation or server
2. PreflightArtifact, storage, media, role checks
3. Create runTask Sequence v2 plan created
4. Boot WinPEVM registers with controller
5. Install WindowsDrivers, setup, OSD client, agent
6. FinishHeartbeat, optional hash/domain/role work

Operational Readiness

The platform reports ready only when there is enough evidence that setup or deployment succeeded.

Stack Ready

Controller healthy, services running, media ready, build host fresh, artifacts promoted.

Workstation Ready

OSDCloud completed, Windows booted, AutopilotAgent heartbeated, optional hash/domain evidence present.

Server Ready

Server image applied, Windows booted, AutopilotAgent online, QGA known, role status ready.

Artifact Ready

Built from source, uploaded with metadata, hashed, and promoted to Proxmox ISO storage.
